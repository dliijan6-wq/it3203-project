Key Concepts

Definitions and explanations of the essential ideas behind PWA development

Service Worker

A JavaScript file that runs in the background separate from the main browser thread. It acts as a network proxy, intercepting requests, managing caches, and enabling offline functionality. Service workers are the core technology that makes PWAs reliable without a network connection.

Web App Manifest

A JSON file that defines how a web application appears when installed on a device. It sets the app's name, icons, theme color, start URL, and display mode. Without a manifest a browser cannot present the web app as an installable standalone experience.

Cache API

A browser storage mechanism that service workers use to store and retrieve network responses. Unlike standard browser caching, the Cache API gives developers full programmatic control over what gets stored and how it gets served back to the user. It is the backbone of offline support in PWAs.

App Shell Architecture

A design pattern where the minimal HTML, CSS, and JavaScript needed to display the application's interface is cached offline. The shell loads instantly from cache while dynamic content loads from the network when available. This gives the impression of an instant launch even on slow connections.

Progressive Enhancement

The design philosophy that a web app should work for every user at a baseline level, then progressively add better functionality for users with more capable browsers. A PWA built on this principle works on old browsers but delivers richer features to users whose browsers support them.

Push Notifications

A feature that allows installed PWAs to send notifications to a user's device even when the app is not open. This requires the user's permission and is handled through the Push API and Notifications API. Push notifications are one of the most impactful PWA features for re-engagement.

IndexedDB

A low-level browser API for storing large amounts of structured data on the client side. PWAs use IndexedDB to store data that needs to persist across sessions and be available offline — like user preferences or data that needs to sync back to a server when the connection is restored. It is more powerful than localStorage but more complex to work with.

How These Concepts Work Together

A fully featured PWA uses all of these concepts in combination. The web app manifest handles installation. The service worker handles network interception and offline caching using the Cache API. The app shell architecture ensures fast first paint. IndexedDB handles persistent local data storage. And push notifications handle re-engagement. The underlying design philosophy — progressive enhancement — ties it together by making sure the app still works when any one capability is unavailable.
