Core Technologies Behind PWAs

The building blocks that make progressive web apps work

Service Workers

Service worker lifecycle: registration, installation, and activation

The service worker is the most important piece of the PWA puzzle. It is a JavaScript file that runs in the background separate from the main browser thread, acting as a programmable network proxy. It can intercept every network request the app makes and decide what to do with it — fetch from the network, return a cached version, or both.

Service workers have a lifecycle with three main stages: registration, installation, and activation. During installation the service worker caches files it needs offline. During activation it takes control of pages under its scope. Once active it intercepts fetch events and applies the caching strategy the developer defined.

Requirements

  • Must be served over HTTPS (localhost is an exception for development)
  • Runs on a separate thread from the main page — cannot access the DOM directly
  • Has its own lifecycle independent of the web page
  • Scope is limited to the directory it is registered from

Web App Manifest

The web app manifest is a JSON file that tells the browser how to display the application when installed on a device. It defines the app's name, icons, theme colors, start URL, and display mode. Without a manifest the browser cannot present the app as a standalone installable experience.

Key Manifest Properties

Property Purpose Example Value
name Full app name shown on install prompt "My PWA App"
short_name Name shown under home screen icon "MyPWA"
start_url Page to open when app is launched "/index.html"
display How the app is presented "standalone"
theme_color Color of the browser toolbar "#1b3a4b"
icons App icons for home screen Array of image objects

HTTPS Requirement

Service workers can only run on pages served over HTTPS. This is a security requirement, not just a recommendation. Because service workers can intercept and modify network requests, running them over HTTP would allow a man-in-the-middle attacker to inject a malicious service worker that intercepts all network traffic.

Free SSL certificates are available through Let's Encrypt, which has made HTTPS adoption much easier. GitHub Pages, which this project is hosted on, serves over HTTPS by default.

Caching Strategies

One of the most important decisions in PWA development is choosing a caching strategy. Different types of content need different approaches depending on how often they update and how critical it is that the user sees the latest version.

Strategy How It Works Best For
Cache First Serve from cache, fall back to network if not cached Static assets: CSS, JS, images
Network First Try network first, fall back to cache if offline Frequently updated content
Stale While Revalidate Serve cache immediately, update in background Content where speed matters more than freshness
Network Only Always fetch from network, no caching Real-time data like stock prices or live scores
Cache Only Always serve from cache, never network Pre-cached offline content
